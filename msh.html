    General .msh Information

    The .msh file format was used by Pandemic Studios for their games using the ZeroEngine(ie: SWBF I+II). Mesh files, with extension *.msh, represent custom 3D models to be placed into new worlds. This file format is used during the munge process, and thus acts as an intermediary between a 3D modelling program and the final *.lvl files used by the game engine.

    Chunk Structure

    The following diagram illustrates how the mesh files hierarchy is organized.

    HEDR
    Mesh file header marker.

    SHVO (exactly one, optional)
    Indicates shadow volume.
    Child of: HEDR

    MSH2 (exactly one)
    Mesh data.
    Child of: HEDR

    SINF (exactly one, optional)
    Scene Information.
    Child of: MSH2
    NAME (exactly one)
    Scene Name.
    Child of: SINF

    FRAM (exactly one)
    Frame range and FPS.
    Child of: SINF

    BBOX (exactly one)
    Bounding box(shown in ZeroEditor).
    Child of: SINF

    CAMR (exactly one, optional)
    Camera information.
    Child of: MSH2
    NAME (exactly one)
    Camera name.
    Child of: CAMR

    DATA (exactly one)
    Camera position.
    Child of: CAMR

    MATL (exactly one)
    Material list.
    Child of: MSH2
    MATD (one or more)
    Material.
    Child of: MATL
    NAME (exactly one)
    Material name.
    Child of: MATD

    DATA (exactly one)
    Material colors(diffuse, ambient, specular) and specular sharpness.
    Child of: MATD

    ATRB (exactly one)
    Material attributes(specular, glow, various renderflags).
    Child of: MATD

    TX0D (zero or one)
    Texture(s).
    Child of: MATD

    MODL (one or more)
    Model.
    Child of: MSH2

    MTYP (exactly one)
    Model type. Indicates if static, dynamic, bone etc.
    Child of: MODL

    MNDX (exactly one)
    Model index number.
    Child of: MODL

    NAME (exactly one)
    Name of model.
    Child of: MODL

    PRNT (optional, exactly one)
    Name of parent model.
    Child of: MODL

    FLGS (optional, exactly one)
    Visibility.
    Child of: MODL

    TRAN (exactly one)
    Matrix transformations.
    Child of: MODL

    GEOM (exactly one, optional)
    Model geometry.
    Child of: MODL

    BBOX (exactly one)
    Model bounding box.
    Child of: GEOM

    SEGM (one or more)
    Mesh segment.
    Child of: GEOM

    SHDW (exactly one)
    Shadow mesh data.
    Child of: SEGM

    MATI (exactly one)
    Index into material list.
    Child of: SEGM

    POSL (exactly one)
    Vertex position list.
    Child of: SEGM

    CLRL (exactly one, optional)
    Vertex colors(colors per node).
    Child of: SEGM

    CLRB (exactly one, optional)
    Vertex colors(one color for segment).
    Child of: SEGM

    WGHT (exactly one, optional)
    Bone, weight percentage pairs list.
    Child of: SEGM

    NRML (exactly one)
    Normal list.
    Child of: SEGM

    UV0L (exactly one, optional)
    UV position list.
    Child of: SEGM

    NDXL (exactly one, optional/deprecated)
    Polygons.
    Child of: SEGM

    NDXT (exactly one, optional/deprecated)
    Triangles.
    Child of: SEGM

    STRP (exactly one)
    Polygon strips.
    Child of: SEGM

    ENVL (exactly one, optional)
    Indices into models(model index).
    Child of: GEOM

    CLTH (exactly one, optional)
    Cloth.
    Child of: GEOM

    CTEX (exactly one)
    Cloth texture.
    Child of: CLTH

    CPOS (exactly one)
    Cloth vertex positions.
    Child of: CLTH

    CUV0 (exactly one)
    Cloth UV positions.
    Child of: CLTH

    FIDX (exactly one)
    Fixed points(indices into CPOS).
    Child of: CLTH

    FWGT (exactly one)
    Cloth fixed point weights.
    Child of: CLTH

    CMSH (exactly one)
    Cloth triangles.
    Child of: CLTH

    SPRS (exactly one)
    Stretch constraint data.
    Child of: CLTH

    CPRS (exactly one)
    Cross constraint data.
    Child of: CLTH

    BPRS (exactly one)
    Bend constraint data.
    Child of: CLTH

    COLL (exactly one)
    Cloth collision.
    Child of: CLTH

    SWCI (exactly one, optional)
    Collision primitive data.
    Child of: MODL

    BLN2 (exactly one)
    Blend factors.
    Child of: HEDR

    SKL2 (exactly one)
    Skeleton.
    Child of: HEDR

    ANM2 (exactly one)
    Animation header.
    Child of: HEDR
    CYCL (exactly one)
    Animation cycles.
    Child of: ANM2

    KFR3 (exactly one)
    Keyframes.
    Child of: ANM2

    CL1L (exactly one)
    Last chunk, never contains data. Still has the length indicator, which is 0.

    Example Chunk


    chunk_name
    Parent: Parent chunk.
    Children: All children chunks.
    Necessity: Once | One or more | Optional
    Description: This is an example.
    See also: Links to further information.
    Additional Information: Every chunk name/header follows a long int indicating the size of the chunk WITHOUT the name/header and size indicator.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int | float | ASCII string | byte
    		number | number each | ~
    		Description.



    Chunks


    HEDR
    Children: SHVO MSH2 BLN2 SKL2 ANM2 CL1L
    Necessity: Once.
    Description: .msh file header.

    MSH2
    Parent: HEDR
    Children: SINF CAMR MATL MODL
    Necessity: Once.
    Description: .msh file mesh header.

    SHVO
    Parent: HEDR
    Necessity: Optional.
    Description: Indicates a shadow volume.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		0 or 1 if .msh has a shadowvolume.

    SINF
    Parent: MSH2
    Children: NAME FRAM BBOX
    Necessity: Optional.
    Description: Scene information header.

    FRAM
    Parent: SINF
    Necessity: Optional.
    Description: Scene frame information
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Start of frame range.
    long int
    		4
    		End of frame range.
    float
    		4
    		Frames per second.

    CAMR
    Parent: MSH2
    Children: NAME DATA
    Necessity: Optional.
    Description: Scene camera header.

    MATL
    Parent: MSH2
    Children: MATD
    Necessity: Once.
    Description: Material list header.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Count of materials(MATD) in this list.

    MATD
    Parent: MATL
    Necessity: One or more.
    Children: NAME DATA ATRB TX0D
    Description: Material header.

    DATA(MATD)
    Parent: MATD
    Necessity: Once.
    Size: 52/60 bytes
    Description: Material colors and specular sharpness.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    float[4]
    		16
    		Diffuse RGBA color(0.0 - 1.0).
    float[4]
    		16
    		Specular RGBA color(0.0 - 1.0).
    float[4]
    		16
    		Ambient RGBA color(0.0 - 1.0).
    float
    		4
    		Specular sharpness.

    ATRB
    Parent: MATD
    Necessity: Once.
    Description: Material flags(glow, rendertypes). See
    See also:
    Inside Edit Flags(GT)
    ZE Flag Calculator
    Render Types
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    byte
    		1
    		Sum of all flag values(check link for more info)
    byte
    		1
    		Render Type number.
    byte
    		1
    		Data0 Value. Range: 0-255
    byte
    		1
    		Data1 Value. Range: 0-255

    TX0D
    Parent: MATD
    Can be extended with: TX1D, TX2D, TX3D.
    Necessity: Optional.
    Description: Texture name with extension(not full path).
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    ASCII string
    		~
    		Texture name.

    MODL
    Parent: MSH2
    Children: MTYP MNDX NAME PRNT FLGS TRAN GEOM SWCI
    Necessity: One or more.
    Description: Model header.

    MTYP
    Parent: MODL
    Necessity: Once.
    Description: Model type.
    Additional Information: 0 = null, 1 = weighted/enveloped mesh, 2 = cloth, 3 = bone, 4 = static mesh, 6 = shadow mesh.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Model type.

    MNDX
    Parent: MODL
    Necessity: Once.
    Description: Model index. 0-based. It's possible to base this on 1, too. Note that you have to take that into account in ENVL etc.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Model index.

    PRNT
    Parent: MODL
    Necessity: Optional.
    Description: Name of parent model, padded with 00's to fill up multiple-of-4 space.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    ASCII string
    		~
    		Model name.

    FLGS
    Parent: MODL
    Necessity: Once.
    Description: Indicates visibility.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Render Flags (valid flags will get added up)
    • 1 Hidden
    • 2 Dynamically Lit
    • 4 Retain Normals
    • 8 RenderAfterShadows
    • 16 Don't Flatten
    • 32 PS2 Optimize

    TRAN
    Parent: MODL
    Necessity: Once.
    Size: 40/48 bytes
    Description: Matrix transform. Scale can get ignored in-game. Usually it should work though. If necessary calculate world space coordinates/let the user Freeze All Transforms before export.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    float[3]
    		12
    		Scale factor in XYZ.
    float[4]
    		16
    		Quaternion Rotation(XYZW).
    float[3]
    		12
    		Translation amount in XYZ.

    GEOM
    Parent: MODL
    Children: BBOX SEGM CLTH ENVL
    Necessity: Optional(depends on MTYP).
    Description: Geometry Header.
    Requirement: MTYP is not 0 or 3.

    SEGM
    Parent: GEOM
    Children: SHDW MATI POSL CLRL CLRB WGHT NRML UV0L NDXL NDXT STRP
    Necessity: One or more.
    Description: Geometry segment header.

    SHDW
    Parent: SEGM
    Necessity: If shadowmesh, once.
    Description: Shadow mesh.
    Structure Example: 4-sided Polygon
    Positions(4)
    0 - (-4.000000,-4.000000,4.000000)
    1 - (-4.000000,-4.000000,-4.000000)
    2 - (-4.000000,4.000000,-4.000000)
    3 - (-4.000000,4.000000,4.000000)

    Edges(8)
    0 - (0, 4, 1, 65535)
    1 - (1, 2, 0, 65535)
    2 - (0, 7, 3, 65535)
    3 - (3, 0, 2, 65535)
    4 - (1, 6, 5, 65535)
    5 - (2, 1, 4, 65535)
    6 - (2, 3, 7, 65535)
    7 - (3, 5, 6, 65535)

    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Number of vertex positions.
    float[3][]
    		12 each
    		Vertex position(X, Y, Z).
    long int
    		4
    		Number of edges.
    short int[4][]
    		8 each
    		Edge. The following 4 entries form one edge.
    | short int
    		2
    		Vertex index of this edge entry. Refers to the vertex list above.
    | short int
    		2
    		Reference into an edge. Defines the target vertex(the local edge vertex of the referenced edge)to which the edge should be drawn from the local vertex.
    | short int
    		2
    		Second reference into an edge. In all example .mshes I've seen this always refers to the same vertex as the first edge reference.
    | short int
    		2
    		MAX_VALUE of short integers(65535). Indicates the end of this edge.


    MATI
    Parent: SEGM
    Necessity: Once?
    Description: Material index(index into MATL).
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Material index.

    POSL
    Parent: SEGM
    Necessity: Once(unless shadowmesh).
    Description: List of vertex coordinates(per node/sample).
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of 3D positions stored in this list.
    float[3][ ]
    		12 each
    		X, Y and Z coordinates for each vertex.

    NRML
    Parent: SEGM
    Necessity: Once(unless shadowmesh).
    Description: List of normal coordinates.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of normal vectors stored in this list.
    float[3][ ]
    		12 each
    		U, V and W coordinates for each vector.

    UV0L
    Parent: SEGM
    Necessity: Optional.
    Description: List of UV coordinates.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of UV positions stored in this list.
    float[2][ ]
    		8 each
    		U and V coordinates for each position.

    CLRL
    Parent: SEGM
    Necessity: Optional.
    Description: List of vertex colors.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of colors per vertex stored in this list.
    byte[4][ ]
    		4 each
    		RGBA colors.

    CLRB
    Parent: SEGM
    Necessity: Optional.
    Description: One vertex color for the complete segment.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    byte[4]
    		4
    		RGBA colors.

    WGHT
    Parent: SEGM
    Necessity: Optional.
    Description: List of bone, weight percentage pairs. There are always 4 pairs per vertex.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of 4 bone/weight pairs stored in this list.
    -
    		-
    		The following appears 4 times per vertex(to allow for soft weights).
    long int
    		4
    		Index into ENVL(which then indexes into MODL=>bone model).
    float
    		4
    		Weight value(0.0 - 1.0).

    NDXL
    Parent: GEOM
    Necessity: Optional/deprecated.
    Description: Polygons. This chunk is deprecated.

    NDXT
    Parent: GEOM
    Necessity: Optional/deprecated.
    Description: Triangles. This chunk is deprecated.

    STRP
    Parent: SEGM
    Necessity: Once(unless shadowmesh).
    Description: List of triangles strips. The start of a strip is indicated by 2 entries with a high bit set(0x8000 or 32768 added). Triangles are listed CW, CCW, CW, CCW... NOTE: In some meshes this chunk has a trailing short which is not calculated into the length/size of this chunk or the # of indices. This short can be ignored. If added to the last polygon it will break it as it always seems to be 0.
    Example:
     Strip values: (13,0,12,32820,32772,20,49)
    Triangles: (13,0,12) (52,4,20) (4,20,49) etc.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of indices into POSL.
    short[ ]
    		2 each
    		Index into POSL. The indices will form polygons.

    ENVL
    Parent: GEOM
    Necessity: Optional, once.
    Description: List of indices into MODL. This model is enveloped to those indices' models. This list can be 1-based.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of indices into MODL.
    long int[ ]
    		4 each
    		Index into a MODL.

    CLTH
    Parent: GEOM
    Necessity: Optional.
    Children: CTEX CPOS CUV0 FIDX FWGT CMSH SPRS CPRS BPRS COLL
    Description: Cloth header, replaces GEOM.

    CTEX
    Parent: CLTH
    Necessity: Once.
    Description: Texture name with extension.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    ASCII string
    		~
    		Texture name.

    CPOS
    Parent: CLTH
    Necessity: Once.
    Description: List of vertex positions(this time per point).
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of vertex positions.
    float[3][ ]
    		12 each
    		XYZ coordinates.

    CUV0
    Parent: CLTH
    Necessity: Once.
    Description: List of texture positions(per point!).
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of vertex positions.
    float[2][ ]
    		8 each
    		UV coordinates.

    FIDX
    Parent: CLTH
    Necessity: Optional?
    Description: List of fixed points.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of points.
    long int[ ]
    		4 each
    		Index into CPOS.

    FWGT
    Parent: CLTH
    Necessity: Optional?
    Description: Index of FIDX weighted to string.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of points.
    ASCII string
    		~
    		Name of the bone the corresponding FIDX is moving with.

    CMSH
    Parent: CLTH
    Necessity: Once.
    Description: Cloth triangles.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of points.
    long int[3][ ]
    		16 each
    		Triangle points defined CCW.

    SPRS
    Parent: CLTH
    Necessity: Optional?
    Description: Stretch constraint data, keeps cloth's size by storing relations between points(like edges around a quad).
    See also: Ande's Post on GT
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of items.
    short[2][ ]
    		4 each
    		Two points sharing an edge.

    CPRS
    Parent: CLTH
    Necessity: Optional?
    Description: Cross constraint data, keeps individual pieces of the cloth from shearing (cross relations between points), think of the edges as sticky tape, glueing the points together.
    See also: Ande's Post on GT
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of items.
    short[2][ ]
    		4 each
    		Two points sharing a quad.

    BPRS
    Parent: CLTH
    Necessity: Optional?
    Description: Bend constraint data, keeps the cloth's shape(relations between border-points of the cloth mesh, disables bending along those edges).
    Possible Values: 0/1 = Sphere; 2 = Cylinder; 4 = Cube.
    See also: Ande's Post on GT
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of items.
    short[2][ ]
    		4 each
    		Two border-points sharing a "straight" edge over the complete cloth.

    COLL
    Parent: CLTH
    Necessity: Optional?
    Description: Cloth collision.
    See also: Ande's Post on GT
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of collisions.
    ASCII string
    		~
    		Collision object name.
    ASCII string
    		~
    		Object name this collision moves with.
    long int
    		4
    		Unknown.
    float
    		4
    		Collision prim data:
    • sphere: radius
    • cylinder: radius
    • cube: length(X)/2
    float
    		4
    		Collision prim data:
    • sphere: radius
    • cylinder: height
    • cube: length(Y)/2
    float
    		4
    		Collision prim data:
    • sphere: radius
    • cylinder: radius
    • cube: length(Z)/2

    SWCI
    Parent: MODL
    Necessity: Optional, once.
    Description: Collision primitive.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		Primitive type.
    • 0: sphere
    • 1: sphere(very rare, the geometry appears to be a sphere, no idea how the engine handles this)
    • 2: cylinder
    • 4: cube
    float
    		4
    		Attribute 0.
    • sphere: radius
    • cylinder: radius
    • cube: sizeX
    float
    		4
    		Attribute 1.
    • sphere: -
    • cylinder: height
    • cube: sizeY
    float
    		4
    		Attribute 3.
    • sphere: -
    • cylinder: -
    • cube: sizeZ

    SKL2
    Parent: HEDR
    Necessity: Optional, once.
    Description: Skeleton/chain. Lists some bone properties for every item in the chain. The values don't seem to make a big difference.
    Structure:
    Data Type
    		 Size(bytes)
    		 Description
    long int
    		4
    		# of bones in this skeleton.
    CRC checksum
    		4
    		CRC checksum of the bone
    long int
    		4
    		Type(bone?). Usual value: 0
    float
    		4
    		Constrain. Usual value: -1.0 or 1.0
    float
    		4
    		Bone length 1. Usual val: 0
    float
    		4
    		Bone length 2. Usual val: 0
